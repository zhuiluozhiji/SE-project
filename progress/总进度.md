## 2. 当前一句话总结

项目已经从“只有 README 和需求文档”初始化成了一个可启动的前后端分离工程。

现在已经具备：

```text
前端 Vue 3 项目可启动
后端 FastAPI 项目可启动
Swagger 接口文档可访问
核心接口已有第一版占位定义
数据库建表 SQL 和种子数据已准备
爬虫、测试、文档目录已创建
各成员的开发入口已经基本明确
```

但也必须注意：

```text
目前还没有接入真实 MySQL 查询
登录还没有真正校验数据库用户
大部分接口返回 mock 数据
前端页面还是骨架和占位组件
爬虫还没有配置真实学院 URL
OCR、ICS、推荐算法还没有正式实现
```

所以当前状态不是“项目做完了”，而是“开发地基已经铺好，大家可以并行开工”。

### 19.6 常见问题

#### 问题 1：`uvicorn: command not found`

说明没有激活虚拟环境，或者没安装依赖。执行：

```bash
source .venv/bin/activate
pip install -r backend/requirements.txt
```

#### 问题 2：`ModuleNotFoundError: No module named 'app'`

启动后端时要带上：

```bash
--app-dir backend
```

正确命令：

```bash
uvicorn app.main:app --app-dir backend --reload --host 0.0.0.0 --port 8000
```

#### 问题 3：前端端口被占用

可以换端口：

```bash
npm run dev -- --host 0.0.0.0 --port 5174
```

#### 问题 4：后端端口被占用

可以换端口：

```bash
uvicorn app.main:app --app-dir backend --reload --host 0.0.0.0 --port 8001
```

如果后端端口改了，前端 `.env` 或 `VITE_API_BASE_URL` 也要同步改。

#### 问题 5：数据库连接失败

优先检查：

```text
MySQL 是否启动
.env 中用户名和密码是否正确
DATABASE_URL 是否正确
schema.sql 是否已经导入
```

当前大部分接口还是 mock 数据，所以数据库连接失败不一定影响接口占位访问，但后续真实业务会依赖数据库。
