### 3.1 系统总体架构

建议写前后端分离架构：

```text
用户浏览器
  ↓
前端页面 Vue/React
  ↓ RESTful API
后端服务 FastAPI/Spring Boot
  ↓
MySQL 数据库
  ↑
爬虫数据导入模块
```

> 系统采用前后端分离架构。前端负责页面展示和交互，后端负责业务逻辑、接口服务、权限控制、冲突检测和 ICS 文件生成，数据库负责存储用户、活动、课表、日程、标签等数据，爬虫模块负责从学院网站采集活动信息并写入数据库。

## 8 协作方式

### 8.1 Git 协作规范

建议仓库结构：

```text
campus-academic-recommender/
├── frontend/
├── backend/
├── crawler/
├── database/
├── docs/
├── tests/
├── README.md

```

这个目录结构和你们之前的开发草案一致。

### 8.2 分支管理规范

```text
main
frontend/activity-list
frontend/calendar-view
backend/auth-api
backend/activity-api
backend/schedule-conflict
backend/recommendation
crawler/cs-zju
docs/development-plan
fix/calendar-bug
```

规则：

* 不直接向 `main` 提交；
* 每个功能开独立分支；
* 合并前本地跑通；
* 涉及接口变化必须通知前端和文档负责人。

### 8.3 接口协作规范

写：

> 接口文档作为前后端联调依据。接口字段一旦进入联调阶段，不随意修改。若必须修改，需要说明修改接口、修改字段、影响页面、数据库是否需要同步变更。

### 8.4 文档同步规范

写：

> `docs/` 目录统一存放接口文档、数据库说明、运行说明、测试记录和答辩素材。每次接口或数据库变更后，相关负责人需要同步更新文档。

### 8.5 会议与沟通机制

写具体一点：

* 每周至少一次组内同步；
* 每次同步记录本周完成内容、当前问题、下周任务；
* 接口变更、数据库变更必须在群内说明；
* 联调问题统一记录到文档或 issue 中。
